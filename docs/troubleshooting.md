# Troubleshooting

## Styles are missing

Import `@gnolith/waystone/styles.css` in the generated root layout. Do not point Tailwind at Waystone source.

## A mutation reports a conflict

Reload the newest entity revision, show the user what changed, and resubmit only after explicit review. Do not remove `expectedRevision` or silently retry.

## An API path returns 404

Align `createWaystoneClient({ paths })` with the routes generated by the installer. Waystone's defaults are conventional, not server implementation leakage.

## A plugin fails during registration

Check lowercase IDs, global contribution-ID uniqueness, and absolute route descriptor paths. Runtime filesystem discovery is unsupported.

## A server-safe route becomes a client boundary

Import static components from `@gnolith/waystone`, not `/client`, `/plugin`, or `/site`. Keep interactive imports in a leaf component with `"use client"`.

## A consuming runtime reports an unsupported import

First reproduce the failure in an isolated consumer using the exact Waystone archive and public import path. Waystone's package verification rejects Node and Cloudflare imports reachable from its public runtime graphs. Complete Site bundle diagnosis, runtime configuration, and deployment acceptance belong to the Codex agent creating the Site; report a Waystone defect only when the isolated package reproduction establishes Waystone causality.
